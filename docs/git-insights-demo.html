Four Git views designed for one project at a time.

This version treats Git as a project-level operating panel. It focuses on one repository: how recently it moved, how much local work is waiting to be pushed, how heavy the current working tree is, and which files have been taking repeated impact.

This is less about branch hygiene and more about commit hygiene. The main question is whether the current tree is still coherent enough to ship cleanly.
Most of the current load is still in tracked modifications, with a smaller tail of new and untracked files. That usually means the repo is in refinement mode rather than broad expansion.
